# BOPE UI Vocabulary Mapping

This document maps default managed-agents platform language into BOPE language.

## Core mappings

- Workspace -> Command Center
- Projects -> Campaigns
- Issues -> Missions
- Inbox -> COMMS
- My Issues -> My Fronts
- Agents -> Battalion
- Runtimes -> Bases
- Skills -> Doctrines
- Settings -> Control

## Agent-specific language

- Assignee -> Active Front
- Blocker -> Tactical Block
- Review -> Validation
- Comment -> COMMS Entry
- Done -> Closed

## Tone rules

- The language should be operational, concise, and evidence-oriented.
- Avoid theatrical excess in daily use.
- Preserve hierarchy where it adds clarity.
- Use doctrine as workflow, not as decorative flavor.

## UX principle

The interface should feel controlled, quiet, and decisive.
It should not feel like a chat toy.
It should feel like a command room with traceable execution.
